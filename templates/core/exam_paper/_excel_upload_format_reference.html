The uploader finds the row where column A is exactly Date of Exam and column B contains Subject. The next row must name your department columns (e.g. SY3, SY_1) above block ranges. Data starts two rows below that (one blank spacer row is normal). Faculty must match short names in column H.

The parser scans the first ~40 rows for headers (case-insensitive). It looks for a Subject column and a Faculty column — accepted faculty headers include: Faculty, Short name, Initial, Evaluator, etc. Date may be labeled Date, Date of duty, … Optional: Notes, Deadline / Due date.

Excel layout — combined supervision (LJIET style)

The uploader finds a row where any cell contains the text Name of Faculty. Three rows directly above that header must be: (1) subject per duty column, (2) exam date per column, (3) time slot per column. Data rows go below the header: faculty full name in column B, initial in column C. Under each duty column, put the division / stream code (e.g. SY_1, SY_3) when that faculty takes the slot — leave blank if not assigned.

3 rows up DM COA TOC
2 rows up 2026-03-30 2026-03-30 2026-04-02
1 row up 10–12 2–4 Morning
Header Sr Name of Faculty Initial Duty columns (same positions as subject row)
Data 1 Anil Kapoor AKS SY_3 SY_3

The parser reads full name from column B and initial from column C on each data row (column A = serial is optional).

  • Division codes in duty cells are normalised (e.g. SY3 → SY_3) to match your sub-units.
  • Rows with “TOTAL” in the name (summary rows) are skipped.
